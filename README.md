# Ming (命/明) - 外物连接NFT仪式平台

> **基于四柱八字能量循环理论的Web3改命平台**  
> 通过NFT作为能量场见证，帮助用户通过外物连接来改命

> **知命明理** - 了解自己的命运，明白其中的道理

---

## 📋 项目简介

### 🎯 核心理念

**"缺失认知，外物连接，NFT见证，仪式完成"**

Ming平台基于传统四柱八字理论，通过分析用户的能量系统循环情况，识别能量缺失，推荐合适的外物进行连接，并通过NFT作为能量场见证，完成改命仪式。

### 💡 核心思想

- **命（先天框架）**: 四柱八字是先天给定的命运框架，不可修改
- **明（知命）**: 通过分析了解自己的能量系统
- **改命（借物改命）**: 通过外物连接，借助公网共识能量场，强行改变命理布局

### 🌟 平台价值

- **仪式感价值**: 提供有意义的象征性连接仪式
- **过程体验价值**: 完整的连接过程体验
- **文化表达价值**: 传统文化元素的现代表达
- **个人意义价值**: 为个人成长创造有意义的时刻

## 🏗️ 技术架构

### 技术栈

- **前端框架**: React 18.2.0 + TypeScript
- **构建工具**: Vite 4.1.0
- **UI组件**: Material-UI 5.11+
- **状态管理**: Redux Toolkit + Zustand
- **区块链**: ethers.js + Solidity
- **存储**: IPFS (Pinata)

## 🚀 快速开始

### 环境要求

- **Node.js**: 18.0+ (推荐 20.x)
- **npm**: 9.0+ 或 **yarn**: 1.22+
- **Git**: 2.30+

### 安装和启动

```bash
# 克隆项目
git clone git@github.com:LuckeeDAO/ming.git
cd ming

# 进入前端目录并安装依赖
cd srcs
npm install

# 启动开发服务器
npm run dev

# 访问应用
open http://localhost:3000
```

### 开发命令

```bash
# 开发环境
npm run dev                    # 启动开发服务器

# 构建项目
npm run build                  # 生产构建
npm run preview                # 预览构建结果

# 代码质量
npm run lint                   # 代码检查
npm run format                 # 代码格式化
npm run type-check             # 类型检查（通过tsc）

# 测试
npm run test                   # 单元测试
npm run test:ui                # 测试UI
npm run test:e2e               # 端到端测试
```

## 🔐 安全与隐私

### 数据安全

- **链上存储**: NFT所有权、连接记录、能量场见证
- **IPFS存储**: 图片、元数据、详细记录
- **本地存储**: 用户临时数据、未发布内容

### 隐私保护

- 四柱八字信息不上链（仅分析结果上链）
- 用户可选择感受记录是否公开
- 本地数据加密存储
