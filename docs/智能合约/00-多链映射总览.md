# 多链映射总览

> 基线版本：v1.0（参照 `docs/00-现行口径基线.md`）

## 1. 文档定位

本文件是智能合约与钱包协议的“多链导航页”，用于说明：

1. 平台协议是多链兼容设计，不绑定单一链；
2. `MING_WALLET_*` 为全链统一消息协议；
3. 各链族在“同一协议字段”之下，分别补充实现映射文档。

## 2. 总体原则

1. 全链协议总口径以 `docs/钱包接口/钱包接口技术规范.md` 为准。
2. 链族选择通过 `contract.chainFamily` 表达（如 `solana` / `evm`）。
3. 链族特有字段（如 Solana `network`、EVM `chainId`）在各链映射文档中细化。
4. 所有链族均应遵循同一安全与验收基线（参数校验、消息幂等、超时处理、权限边界）。

## 3. 链族映射入口

### 3.1 Solana（当前已接入）

- 设计文档：`docs/智能合约/01-Solana智能合约设计.md`
- 指令映射：`docs/智能合约/02-钱包协议与Solana指令映射.md`
- 安全分析：`docs/智能合约/03-Solana合约安全性分析与修正.md`
- 代码目录：`contracts/solana`

### 3.2 EVM（预留入口）

- 当前状态：占位文档已建立，待按相同协议字段补充 EVM 专项映射细节。
- 占位文档：`docs/智能合约/04-EVM映射占位文档.md`
- 约束要求：
  1. 必须复用 `MING_WALLET_*` 协议；
  2. 必须遵守 `protocolVersion=1.0.0` 与 `timing` 字段要求；
  3. 必须在文档中明确 `chainId`、地址校验、交易回执与业务标识映射规则。

## 4. 版本策略

当前为首版协议（v1.0），不引入升级迁移叙事；新增链族时采用“新增子文档，不改总口径”的方式演进。
